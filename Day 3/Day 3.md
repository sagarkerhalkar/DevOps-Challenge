 2. Service Checks
Inspecting the state of the OpenSSH server (ssh) systemd service.

      Command 3:    systemctl status ssh 
          Purpose:    Displays the full operational runtime status, uptime, and recent logs of the SSH service.  
          Sample Output:     
bash
        ● ssh.service - OpenBSD Secure Shell server

      Command 4:    systemctl is-active ssh
          Purpose:    Returns a simple, single-word verification of whether the service is currently running.

4. Mini Troubleshooting Flow
   Scenario:    SSH service is reported as unreachable.

1.     Verify Service Status:    `systemctl is-active ssh` returned `inactive`.
2.     Check Logs for Errors:    Ran `journalctl -u ssh -n 20` to pinpoint configuration issues.
3.     Attempt Resolution:    Fixed a minor config error and applied changes using:  
    `sudo systemctl restart ssh`
4.     Confirm Success:    Verified the service successfully transitioned back to `active (running)`.
